Software Development Principles

Should Be Embraced Daily Being a good programmer is a mix of skills and common sense.

YAGNI - You Aren't Gonna Need It

  • Add code that is needed right away.
  • Refactor and remove unused code.
  • Embrace version control for code retrieval.

Examples:

function add(a, b) {
  // Unnecessary check that we aren't gonna need
  if (typeof a !== 'number' || typeof b !== 'number') {
    return null;
  }
  // Unnecessary variable declaration
  const result = a + b;

  // Unnecessary logging
  console.log(`The result is: ${result}`);

  return result;
}


  function add(a, b) {
  // Simpler version that follows YAGNI
  return a + b;
}

DRY - Don't Repeat Yourself

  • Maintain a single source of truth for information and data.
  • Avoid duplicated code for better maintenance.
  • Reuse existing code when possible.

KISS - Keep It Simple, Silly

  • Simpler systems work best and more reliably.
  • Avoid overengineering and using new tools unnecessarily.
  • Choose the simplest and most efficient solutions.

Examples:

  function calculateGrade(score) {
    // Bad: Overly complex code that violates KISS
    if (score >= 90) {
        return "A";
      } else if (score >= 80) {
        return "B";
      } else if (score >= 70) {
        return "C";
      } else if (score >= 60) {
        return "D";
      } else {
        return "F";
      }
  }


  function calculateGrade(score) {
    // Good: Simpler and adhering to KISS
    if (score >= 90) return "A";
    if (score >= 80) return "B";
    if (score >= 70) return "C";
    if (score >= 60) return "D";
    return "F";
  }

Big Design Up Front thoroughly before implementation

  • Avoid rushing into coding without a clear plan.
  • Involve others in the design process for better outcomes.

Avoid Premature Optimization

  • Don't optimize until necessary.
  • Apply KISS and YAGNI principles.
  • Premature optimization can lead to delays and increased costs.

  • Conclusion - Embrace these principles for better programming.
  • Increase awareness and apply one principle at a time.
  • Simplicity and power make these principles valuable.
